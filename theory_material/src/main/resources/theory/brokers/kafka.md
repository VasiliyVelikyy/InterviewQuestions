### Топики кафки

Топики в кафке зависимы от регистра.
```TOPIC_FOR_CONSUMER ``` и ```topic_for_consumer``` два разных топика

### Несколько МС на один топик

Имеется 2 микросервиса со слушателями на 1 топик. Если groupId
Разные groupId получат сообщения все. То есть микросервис 1 получит и 2 одно и тоже сообщение.
Но если есть консьюмеры внутри в одного groupid. То сообщение получит один слушателей из всех. Остальные будут рабоатьь
в холостую

### Правила генерации groupId для слушателя в МС на springboot

если в микросервисе на Spring Boot не указан groupId для потребителя (consumer),
то Spring Boot Kafka автоматически сгенерирует groupId на основе следующих правил:

- Имя приложения (spring.application.name):
    - Если в конфигурации Spring Boot указано свойство spring.application.name, то оно будет использовано как часть
      groupId.
    - Например, если ```spring.application.name=my-service```, то groupId может быть сгенерирован как my-service-group.

- Имя бина потребителя:
    - Если spring.application.name не указано, Spring Boot может использовать имя бина потребителя Kafka в качестве
      groupId.

- Случайное значение:
    - Если никакие другие параметры не заданы, Spring Boot может сгенерировать случайное значение для groupId.

Пример конфигурации
Если вы используете Spring Boot Kafka и не указываете groupId явно, это может выглядеть так:

```yaml
spring:
  application:
    name: my-service
  kafka:
    consumer:
      auto-offset-reset: earliest
```


В этом случае groupId может быть сгенерирован как ```my-service-group```

#### Как указать groupId явно

Чтобы избежать неожиданного поведения, рекомендуется всегда указывать groupId явно в конфигурации:

```yaml
spring:
  kafka:
    consumer:
      group-id: my-custom-group-id
```

Если groupId не указан, это может привести к тому, что каждый экземпляр приложения будет использовать уникальный
groupId, что может вызвать проблемы с обработкой сообщений (например, каждое сообщение будет обрабатываться всеми
экземплярами).

Убедитесь, что groupId одинаков для всех экземпляров микросервиса, которые должны работать в одной потребительской
группе.

Если вы хотите точно знать, как groupId генерируется в вашем случае, проверьте логи приложения при старте — Spring Boot
обычно выводит информацию о конфигурации Kafka, включая groupId.
